<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser navigation → discovery page | User can request discovery route and inspect rendered household data |
| Discovery template rendering → recipe/inventory metadata | Untrusted display context could leak cross-household inventory state if scope checks fail |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | I | DiscoveryView context assembly | mitigate | Require `LoginRequiredMixin` and call service with `request.user.household` only |
| T-04-05 | S | /shopping/discovery/ endpoint access | mitigate | Enforce authentication redirect for anonymous users and keep URL within shopping auth-protected namespace |
| T-04-06 | T | template conditional urgency badges | mitigate | Render urgency strictly from server-computed flags (`has_urgent_match`, `has_expired_match`) not query params |
</threat_model>

# Phase 2: AI Service & API Integration

## Goal
Create the AI service layer that generates weekly meal plans by calling opencode.ai. Build prompt construction, API integration with retry logic, response parsing, and wire it into the planner UI with a "Generate AI Plan" button.

**Dependencies:** Phase 1 complete (MealPreferences model exists)

---

## Tasks

### Task 1: Create services directory and AIService class

**Files:** `meal_planner_app/services/__init__.py`, `meal_planner_app/services/ai_service.py`

- Create `meal_planner_app/services/` Python package
- `AIService` class with:
  - `base_url = "https://opencode.ai/zen/v1"` (configurable via setting)
  - `model = "opencode/deepseek-v4-flash-free"` (configurable via setting)
  - `generate_meal_plan(houshold, start_date, end_date)` main method
  - Prompt builder: constructs structured prompt from preferences + inventory + date range
  - Retry logic with exponential backoff (max 3 attempts, 1s/2s/4s)
  - Error handling: returns `AIServiceResult` with success/error/parsed_meals fields

**Prompt design:**
- System prompt: role as creative meal planning chef
- User prompt includes:
  - Cuisine preferences
  - Dietary restrictions
  - Cooking effort preference
  - Servings per meal
  - Excluded ingredients
  - Available inventory items (especially near-expiry)
  - Date range (N days)
- Output format instruction: strict JSON array with per-day structure

**Requirements:** GENR-03, GENR-04, GENR-05, GENR-06, GENR-09, GENR-11

### Task 2: Create Response Parser

**Files:** `meal_planner_app/services/response_parser.py`

- `parse_weekly_plan(ai_response_text)` function
- Extracts JSON array from response (handles markdown code blocks, extra text)
- Returns list of daily meal plans with: date, meal_type, title, description, cook_time, ingredients
- Validation: all days covered, all required fields present
- Fallback on parse failure

**Requirements:** GENR-07, GENR-08, GENR-10

### Task 3: Add generation view

**Files:** `meal_planner_app/views.py`

- `GenerateAiPlanView(LoginRequiredMixin, View)` — POST only
- POST flow:
  1. Get week start/end from POST data
  2. Load user's MealPreferences
  3. Load household inventory (non-expired items)
  4. Call `AIService.generate_meal_plan()`
  5. On success: create MealPlan entries for each generated meal
  6. On error: show error message
  7. Redirect back to planner page with messages

**Requirements:** GENR-01, GENR-02, GENR-08

### Task 4: Add URL route

**Files:** `meal_planner_app/urls.py`

- `path("generate-ai-plan/", views.GenerateAiPlanView.as_view(), name="generate_ai_plan")`

### Task 5: Add "Generate AI Plan" button to planner

**Files:** `meal_planner_app/templates/meal_planner/planner.html`

- Add button in the planner toolbar section
- POST form to `{% url 'meal_planner:generate_ai_plan' %}` with hidden week_start input
- Loading/disabled state while generating
- Confirm dialog ("This will replace existing meals for this week?")

**Requirements:** GENR-01

### Task 6: Tests

- `AIServiceTests`:

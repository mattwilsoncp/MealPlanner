# Phase 3: AI Plan Review & Acceptance

## Goal
Build the interactive review workflow for AI-generated meal plans. Replace the current auto-save behaviour (GenerateAiPlanView saves MealPlan entries immediately) with a **preview → accept/regenerate/reject → save** flow where the user reviews and selectively commits AI suggestions.

## Architecture

### Current Flow (Phase 2)
```
[AI Plan button] → POST generate-ai-plan → AIService → creates MealPlan entries → redirects to planner
```

### Target Flow (Phase 3)
```
[AI Plan button] → POST generate-ai-plan → AIService (no save) → store in session → redirect to review page

[Review page] shows 7 days with controls per day:
  ├── Accept ✅ — marks day for saving
  ├── Reject ✕ — removes day from preview
  └── Regenerate 🔄 — calls AIService for that single day, replaces in preview

[Save Plan] → creates MealPlan entries for accepted days only → redirects to planner
[Cancel] → clears session → redirects to planner
```

### Session Storage
Use Django session to hold the pending plan:
- **Key:** `ai_pending_plan:{household_id}:{year}-{week}` — ensures isolation per household/week
- **Value structure:**
  ```json
  {
    "week_start": "2026-06-01",
    "days": [
      {
        "date": "2026-06-01",
        "status": "pending|accepted|rejected",
        "meals": [
          {"meal_type": "breakfast", "title": "...", "description": "...", "cook_time_minutes": 30, "ingredients": [...]},
          ...
        ]
      },
      ...
    ]
  }
  ```

### Design Decisions

| Decision | Rationale |
|----------|-----------|
| Session storage over DB temp model | Simpler, no migration, auto-expires with session. Plan lives only during review. |
| Per-day granularity (not per-meal) | REVW-01/02/03/04 all describe day-level operations. Simplifies UI + logic. |
| Regenerate replaces day in-place | Keeps the week intact. User can keep regenerating until satisfied. |
| Rejected days removed from preview | Clean visual — user sees only what might be saved. Matches REVW-04. |
| Existing slot check on save (not preview) | REVW-07: "Don't overwrite existing entries" applies at save time. Preview shows empty/occupied state. |
| Session key includes household + week | Prevents conflicts if user opens multiple weeks. |
| Single-page review UI | No multi-step wizard — all days visible at once with inline controls. |

### Task 2: Create AiPlanReviewView

**Files:** `meal_planner_app/views.py`

- `AiPlanReviewView(LoginRequiredMixin, View)` — GET only
- Reads session data for `ai_pending_plan:{household.pk}:{week_start}` (week_start from GET param)

- Renders `meal_planner/ai_plan_review.html` with:

  - `week_start` — for navigation and save/cancel URLs

  - `week_dates` — formatted for display

**Requirements:** REVW-01

### Task 3: Create AiPlanDayActionView — accept/regenerate/reject

**Files:** `meal_planner_app/views.py`

- `AiPlanDayActionView(LoginRequiredMixin, View)` — POST only
- Single view handling three actions, differentiated by `action` POST parameter:
  - `accept` — Sets `day["status"] = "accepted"`
  - `reject` — Sets `day["status"] = "rejected"`
  - `regenerate` — Calls `AIService.generate_meal_plan()` scoped to a SINGLE day (reuse existing service with start_date=end_date for that day), replaces the day's meals in session, sets status back to `pending`

**Parameters:** `action` (POST), `day_index` (POST, index into days array)

**Regenerate detail:**
- Call `AIService.generate_meal_plan()` with `start_date=end_date=target_date`
- Parse and validate the single day result
- Replace the existing day's meals in session with the new result
- Keep the same date, reset status to `pending`
- On failure, keep old data and add error message

After any action, save session and return redirect back to the review page.

**Requirements:** REVW-02, REVW-03, REVW-04

### Task 4: Create AiPlanSaveView — commit to MealPlan

**Files:** `meal_planner_app/views.py`

- `AiPlanSaveView(LoginRequiredMixin, View)` — POST only
- Reads session data for `ai_pending_plan:{household.pk}:{week_start}`
- Iterates days with `status == "accepted"`
- For each day, for each meal in the day:
  1. Check slot is not already occupied: `MealPlan.objects.filter(household=household, meal_date=date, meal_type=meal_type).exists()`
  2. If slot is empty: create `MealPlan(household=household, meal_date=date, meal_type=meal_type, custom_meal=f"{title}: {description}"[:500], notes=f"AI-generated | Cook time: {cook_time} min")`
  3. If slot is occupied: skip (maintains REVW-07)
- After saving, clear the session key
- Show success message with count of meals created
- Redirect to `planner_week`

**Edge cases:**
- No accepted days: show info, redirect back to review
- All slots already filled: show info, no duplicates created
- Session expired/tampered: error message, redirect to planner

**Requirements:** REVW-05, REVW-06, REVW-07

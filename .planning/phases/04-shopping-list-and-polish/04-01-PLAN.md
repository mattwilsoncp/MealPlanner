# Phase 4: Shopping List & Polish

## Goal
Connect AI-generated meals to the shopping list. AI meals save without structured ingredient data, so the existing `generate_week_shopping_list` (which looks up ingredients via `IngredientLink` → `Recipe`) skips them entirely. We need to persist AI meal ingredients, extend the shopping list generator to handle AI meals, and wire up navigation so the user can get from AI plan save → shopping list.

## Architecture

### Current State
```
AI plan save (AiPlanSaveView)
  → MealPlan.recipe = NULL
  → MealPlan.custom_meal = "Title: Description"
  → MealPlan.notes = "AI-generated | Cook time: N min"
  → ingredients from session are DISCARDED ✗

generate_week_shopping_list (shopping/services.py:144)
  → MealPlan.objects.filter(recipe__isnull=False)  ← AI meals excluded
  → Looks up IngredientLink via recipe_id
  → AI meals (no recipe) produce no shopping items ✗
```

### Target State
```
AI plan save (AiPlanSaveView)
  → MealPlan.recipe = NULL
  → MealPlan.custom_meal = "Title: Description"
  → MealPlan.ingredients = ["oats", "milk", "banana"]  ← NEW: JSONField

generate_week_shopping_list (shopping/services.py:144)
  → For recipe-based meals: same as before (IngredientLink → Recipe)
  → For AI meals (recipe=NULL + ingredients populated):
    - Parse each ingredient name from JSONField
    - Check inventory by name match (same normalize_unit_key logic)
    - Create ShoppingListItem for missing ingredients
    - No unit/quantity on AI ingredients (they're just names) — default qty=1, unit="piece"
```

### Data Flow
```
[Save AI Plan] → saves MealPlan with ingredients JSON → redirects to planner
                                              ↓
                                    planner template shows
                                    "N meals saved! View Shopping List" link
                                              ↓
                              [Shopping Week view] → generate_week_shopping_list
                                                       picks up AI meals too
```

- JSONField with `default=list` creates a new empty list per instance (safe from mutable default issues because Django JSONField handles this)

### Task 2: Modify AiPlanSaveView to persist ingredients
**File:** `meal_planner_app/views.py`

In the `post()` method of `AiPlanSaveView` (~line 1248), add `ingredients` to the MealPlan creation call:

```python
ingredients_list = meal_data.get("ingredients", [])
MealPlan.objects.create(
    household=household,
    meal_date=meal_date,
    meal_type=meal_type,
    custom_meal=f"{title}: {description}".strip(": ")[:500],
    notes=f"AI-generated | Cook time: {cook_time} min",
    ingredients=ingredients_list,       # ← NEW
)
```

**Edge cases:**
- `ingredients` may be missing from session data (old sessions from before this change) → default `[]`
- Ingredients may be None/null → treat as empty list
- Ingredients may contain duplicates → handle in shopping list generation, not here

### Task 3: Extend generate_week_shopping_list to handle AI meals
**File:** `shopping/services.py`

Modify `generate_week_shopping_list` to:

1. Remove the `recipe__isnull=False` filter — fetch ALL meal plans for the week
2. For `recipe__isnull=False` meals (recipe-based): keep existing IngredientLink logic
3. For `recipe__isnull=True` meals with `ingredients` populated (AI meals):
   - Iterate each ingredient name
   - Normalize with `normalize_unit_key(name, "piece")`
   - Check against `remaining_inventory` using the same gram-based dedup
   - If needed > 0 after inventory dedup, create a ShoppingListItem with qty=1, unit="piece", category="other"
   - Set `source_recipe=None` for AI meal items

**Edge cases:**
- AI meal with empty ingredients list → skip
- Ingredient name already matched by inventory → no shopping item created (SHOP-03)
- Same ingredient across multiple AI meals → should aggregate (eat inventory then dedup across meals)
- AI ingredient "butter" vs inventory "Butter (salted)" — `normalize_unit_key` handles case-folding but not parenthetical variants. Accept this limitation for now (Phase 4 scope).

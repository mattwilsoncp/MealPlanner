<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser checklist actions → shopping mutation endpoints | Untrusted action payload can modify/delete shopping records |
| Shared navigation links → week query parameters | User-controlled parameters can request other households' data |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-13 | E | toggle/delete/clear endpoints | mitigate | Enforce household ownership on every ShoppingListItem/ShoppingListWeek lookup before mutation |
| T-03-14 | T | JS action payloads | mitigate | Require CSRF token in fetch POST requests and reject non-POST mutating operations |
| T-03-15 | R | destructive clear action | mitigate | Return explicit JSON result with affected count to support auditability in logs/tests |
</threat_model>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser filter/query params → inventory queryset | Untrusted query parameters control DB filtering |
| Browser POST/DELETE actions → inventory mutations | User input can attempt cross-household edits/deletes |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | I | inventory list/search | mitigate | Restrict all list/queryset/filter queries to `request.user.household` before applying user filters |
| T-03-05 | T | quick-add API | mitigate | Validate payload through ModelForm and keep CSRF enabled (remove csrf_exempt) |
| T-03-06 | E | edit/delete views | mitigate | Use `get_object_or_404(..., household=request.user.household)` for updates/deletes |
</threat_model>
